# 故障排查

本文档用于处理 Qproxyhub 常见启动和运行问题。

---

## 1. 双击后没有打开网页

请依次检查：

1. 本机是否已安装 Node.js  
默认路径：
`C:\Program Files\nodejs\node.exe`

2. 本机是否已准备好可用的 `mihomo` 内核  
默认路径：
`C:\Program Files\mihomo-windows-amd64-v1.19.24\mihomo-windows-amd64.exe`

3. `8866` 端口是否被其他程序占用

---

## 2. PowerShell 提示 `.\start-proxy-tester.ps1` 不存在

这是因为你不在项目目录里执行。

正确方式：

```powershell
cd D:\Xcode\20260423_Qproxyhub
powershell -ExecutionPolicy Bypass -File .\start-proxy-tester.ps1
```

或者直接使用：

- `Qproxyhub.exe`
- `Qproxyhub-启动.bat`

---

## 3. 页面能打开，但一直提示服务未连接

通常说明本地服务没有成功启动。

优先检查：

- Node.js 是否安装
- `start-proxy-tester.ps1` 是否被安全策略拦截
- 是否有旧进程占用了端口

---

## 4. 为什么批量测试结果比单条手测差很多

常见原因是：

- 超时设置太低
- 并发设置太高
- 同一服务商节点被同时压测

推荐参数：

- 超时：`8 秒` 或 `12 秒`
- 并发：`1 条（更稳）`

---

## 5. `.exe` 能不能脱离其他文件单独运行

当前不行。

`Qproxyhub.exe` 目前是启动器，不是完整打包版。
它需要与这些项目文件放在同一目录或完整目录结构中：

- `server.mjs`
- `web/`
- `start-proxy-tester.ps1`

如果你是普通使用者，建议优先下载：

- `Qproxyhub-v0.1.0-portable.zip`

---

## 6. 为什么 Release 里既有 `.exe` 又有 `.zip`

区别是：

- `.exe`：适合已经拿到完整目录的人，用来直接启动
- `.zip`：适合直接下载后解压使用，内容更完整

---

## 7. 某些节点上次通过，这次失败，是不是工具有问题

不一定。

代理节点本身存在波动，尤其是：

- 短时 IP
- 同一供应商的一批同源节点
- 高并发压测时

如果你怀疑是假失败，建议：

1. 改成单并发
2. 提高超时
3. 对失败项单独复测
