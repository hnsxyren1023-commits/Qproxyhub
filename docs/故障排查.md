# 故障排查

本文档用于处理 Qproxyhub 的常见启动与运行问题。

---

## 1. 双击后没有打开网页

请按顺序检查：

1. Node.js 是否安装  
默认路径：`C:\Program Files\nodejs\node.exe`

2. mihomo 内核是否存在  
默认路径：`C:\Program Files\mihomo-windows-amd64-v1.19.24\mihomo-windows-amd64.exe`

3. `8866` 端口是否被其他程序占用

---

## 2. PowerShell 提示 `.\start-proxy-tester.ps1` 不存在

原因通常是当前目录不在项目目录。

正确方式：

```powershell
cd D:\Xcode\20260423_Qproxyhub
powershell -ExecutionPolicy Bypass -File .\start-proxy-tester.ps1
```

或者直接使用：

- `Qproxyhub.exe`
- `Qproxyhub-启动.bat`

---

## 3. 页面能打开，但显示“服务未连接”

说明本地服务没有成功启动，重点检查：

- Node.js 是否正常安装
- `start-proxy-tester.ps1` 是否被执行策略拦截
- 是否有旧进程占用了端口

---

## 4. 批量测试和单条直测结果差异较大

常见原因：

- 超时设置过低
- 并发设置过高
- 同一供应商节点被并发压测

建议参数：

- 超时：`8 秒` 或 `12 秒`
- 并发：`1 条（更稳）`

---

## 5. `Qproxyhub.exe` 能否单独运行

当前不能完全脱离项目文件单独运行。  
`Qproxyhub.exe` 目前是启动器，需要与以下文件保持同目录结构：

- `server.mjs`
- `web/`
- `start-proxy-tester.ps1`

普通用户建议优先下载：

- `Qproxyhub-v0.1.0-portable.zip`

---

## 6. 为什么 Release 里既有 `.exe` 又有 `.zip`

- `.exe`：适合已经有完整项目目录的用户
- `.zip`：适合直接下载解压即用，内容最完整

---

## 7. 节点上次通过这次失败，是不是工具有问题

不一定。代理节点本身存在波动，特别是：

- 短时 IP
- 同源节点池
- 高并发压测场景

建议使用“失败重试”或“历史复测”进行二次确认。
