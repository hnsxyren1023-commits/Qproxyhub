# 故障排查

本页用于处理 Qproxyhub 常见启动与运行问题。

## 1. 双击后没有打开网页

请按顺序检查：

1. Node.js 是否安装  
默认路径：`C:\Program Files\nodejs\node.exe`
2. mihomo 内核是否存在  
默认路径：`C:\Program Files\mihomo-windows-amd64-v1.19.24\mihomo-windows-amd64.exe`
3. 端口 `8866` 是否被占用

## 2. PowerShell 提示 `.\start-proxy-tester.ps1` 不存在

这是因为你当前目录不在项目目录。

正确方式（任意目录都可以执行）：

```powershell
powershell -ExecutionPolicy Bypass -File "D:\Xcode\20260423_Qproxyhub\start-proxy-tester.ps1"
```

或直接双击：

- `Qproxyhub.exe`
- `Qproxyhub-启动.bat`

## 3. 页面能打开，但显示“本地服务未连接”

说明本地服务没有成功启动，请检查：

- Node.js 是否正常安装
- 是否被执行策略拦截
- 是否已有旧进程占用端口

## 4. 批量测试和单条直测差异较大

常见原因：

- 超时设置过低
- 并发设置过高
- 节点本身波动

建议参数：

- 超时：`8秒` 或 `12秒`
- 并发：`1`

## 5. `.exe` 与 `.zip` 的区别

- `.exe`：单文件启动器，适合已有完整目录结构的用户
- `.zip`：便携完整包，下载解压即可用，推荐普通用户使用

## 6. 节点上次通过这次失败，是不是工具有问题

不一定。代理节点天然存在波动，尤其是短时 IP、同源节点池与高并发场景。  
建议使用“重试失败项”与“历史复测”进行二次确认。
