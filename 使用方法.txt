Proxy Link Tester 使用方法

1. 启动方式
在 PowerShell 中运行：

powershell -ExecutionPolicy Bypass -File D:\Xcode\20260423_Qproxyhub\start-proxy-tester.ps1

启动成功后，终端会显示：
Proxy tester is running at http://127.0.0.1:8866/

2. 打开页面
浏览器访问：
http://127.0.0.1:8866/

如果你之前开过旧版页面，请按 Ctrl + F5 强制刷新一次。

3. 支持的链接类型
- socks5://...
- http://...
- hy2://...
- vless://...
- vmess://...

4. 基本测试流程
- 将代理链接按“每行一条”粘贴到输入框
- 选择超时时间
- 选择并发数量
- 点击“开始测试”
- 页面会实时显示：
  当前阶段
  整体进度
  当前正在测试的节点
  通过、失败、无效数量
  每条结果的延迟、账号、密码、说明

5. 结果表功能
- 点击表头排序
- 按关键字筛选
- 勾选单条或多条
- 复制选中链接
- 复制选中行
- 选中当前可见结果
- 清空选择

6. 历史任务
- 页面右下角显示历史任务列表
- 点击某条历史任务，可以重新加载它的结果快照
- 历史记录会保存在：
  D:\Xcode\20260423_Qproxyhub\exports\history.json

7. 失败重试
- “重试失败项”：
  只对当前任务或已选历史任务中的失败节点重新测试
- “重新测试全部”：
  对当前任务或已选历史任务中的全部节点重新测试

8. 导出中心
导出前可以选择：
- 来源：当前任务 / 历史任务
- 范围：仅通过 / 仅失败 / 全部 / 仅选中
- 格式：TXT / CSV / JSON / 客户端配置

当前支持的客户端导出：
- mihomo
- Clash Verge
- v2rayN
- NekoBox / Nekoray

说明：
- mihomo、Clash Verge 会导出 YAML 配置
- v2rayN、NekoBox / Nekoray 目前导出为原始链接集合，方便直接导入

9. 导出文件位置
所有测试结果和导出文件默认保存在：
D:\Xcode\20260423_Qproxyhub\exports

10. 目录说明
- start-proxy-tester.ps1：启动脚本
- server.mjs：本地服务与测试逻辑
- web\：前端单页面
- exports\：CSV、JSON、TXT、YAML、历史记录
- temp\：运行时临时文件

11. 关闭方式
- 在启动服务的 PowerShell 窗口中按 Ctrl + C
- 或直接关闭该 PowerShell 窗口

12. 补充说明
- 这是本地工具，页面只是前端界面
- 真正的连通性测试由本地 mihomo 内核完成
- 如果 8866 端口已被占用，请先关闭旧进程后再启动
- 如果你在测同一服务商的一批节点，优先使用：
  8 秒或 12 秒超时
  1 条并发
  这样更接近人工单条直测结果
